## 🧠 상세 내용 정리

### 1. OAuth(Open Authorization)란?
- 정의 : 사용자가 비밀번호를 직접 제공하지 않고
        제3자 애플리케이션이 사용자의 리소스에
        접근할 수 있도록 권한을 위임하는 개방형 프로토콜
- 구분 : 인가(Authorization) - 권한
- 토큰 : Access Token
- 장점
    (1) 비밀번호 노출 없이 안전하게 권한 위임 가능
    (2) 권한 범위(scope) 지정이 유연
    (3) 토큰 만료, 폐기, 권한 관리가 용이
- 단점
    (1) 인증(Authentication) 기능이 부족
    (2) 구현과 관리가 복잡할 수 있음
    (3) 인증(Authentication) 용도로만 사용할 경우 보안상 결함 발생 가능

### 2. OIDC(OpenID Connect)란?
- 정의 : OAuth 2.0을 기반으로 한
        인증(Authentication) 프로토콜
- 구분 : 인증(Authentication) - 신원
- 토큰 : Access Token + ID Token
- 장점
    (1) ID Token(JWT)에 표준화된 사용자 정보 포함 -> 별도 API 호출 없이 인증 및 기본 정보 획득 가능
    (2) 다양한 인증 시나리오에 적합
    (3) JWT 기반으로 구현이 간편하고, 확장성 및 상호운용성 높음
- 단점
    (1) OAuth 단독 대비 구현 복잡도 증가
    (2) ID Token 검증 및 관리에 대한 학습 장벽 높음
    (3) 설정 및 운영 난이도 높음
