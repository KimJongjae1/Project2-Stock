## 🧠 상세 내용 정리

### 1. 동기(Synchronous)란?
- 정의 : 요청한 작업이 끝날 때까지 결과를 기다리며
        작업이 완료되어야 다음 단계를 진행할 수 있는 방식
- 특징 : 작업 순서가 항상 보장됨
- 장점
    (1) 직관적이고 구현이 간단함
    (2) 실행 순서 예측이 가능함
- 단점
    (1) 작업이 진행되는 동안 기다려야 하므로 자원 활용이 비효율적임
    (2) 대기시간 동안 다른 작업을 처리하지 못해 전체 성능이 저하됨

### 2. 비동기(Asynchronous)란?
- 정의 : 작업을 요청한 후 결과를 기다리지 않고
        바로 다음 작업을 수행하는 방식
- 특징 : 이벤트 루프, 콜백 함수 등을 활용함
- 장점
    (1) 여러 작업을 동시에 처리가 가능함
    (2) 자원을 효율적으로 사용해서 성능과 반응성 향상됨
- 단점
    (1) 설계 및 구현이 복잡하고 오류 관리가 어려움
    (2) 코드 가독성 저하, 순서 보장 어려움

### 3. 블로킹(Blocking)?
- 정의 : 어떤 작업이 완료될 때까지
        현재 프로그램 실행 흐름이 멈추는 방식
- 특징 : 결과가 반환될 때까지 기다림
- 장점
    (1) 구조가 단순하고 직관적임
    (2) 디버깅이나 예측이 쉬움
- 단점
    (1) 자원 활용률이 떨어지고, 대기 시간이 길어질 수 있음
    (2) 데드락 등 병목 현상이 발생 가능함

### 4. 논블로킹(Non-Blocking)란?
- 정의 : 작업을 요청하더라도 현재 작업이 멈추지 않고
        바로 다음 작업을 이어서 수행하는 방식
- 특징 : 요청 후 곧바로 제어권 반환함
- 장점
    (1) 다수의 작업을 병렬로 처리 가능함
    (2) 시스템 성능 및 자원 활용 극대화가 가능함
- 단점
    (1) 결과 처리 코드가 복잡해질 수 있음
    (2) 작업 순서, 일관성 관리 어려움
