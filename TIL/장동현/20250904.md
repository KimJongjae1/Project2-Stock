## 🧠 상세 내용 정리

### 1. 추상화(Abstraction)란?

- 정의 : 불필요한 세부 사항을 숨기고 핵심적인 공통 속성과 행위만 추출하는 것
- 특징 : 현실 세계를 모델링할 때 복잡도를 줄이고
        중요한 정보만 프로그램에 적용하도록 설계
- 장점
    (1) 복잡한 시스템의 단순화
    (2) 명확한 설계 제공

### 2. 다형성(Polymorphism)란?

- 정의 : 동일한 메서드에 대해 여러 형태로 동작할 수 있게 하는 것
- 특징 : 오버라이딩, 오버로딩으로 인한 메서드 사용법
- 장점
    (1) 프로그램 확장성 좋음
    (2) 프로그램 유연성 좋음

### 3. 상속(Inheritance)란?

- 정의 : 기존 클래스(부모)의 속성과 기능을 새로운 클래스(자식)에게 물려주는 것
- 특징 : 계층적 구조를 통해 공통된 기능 제공
- 장점
    (1) 코드 중복 최소화
    (2) 유지보수성 향상

### 4. 캡슐화(Encapsulation)란?

- 정의 : 정보 은닉의 한 방법으로 데이터와 메서드를 하나의 단위로 묶고
        외부에서 객체의 내부 상태에 직접 접근하지 못 하게 하는 것
- 특징 : 객체 외부에서는 필요한 메서드만 공개하고 내부 구현은 감춤
- 장점
    (1) 데이터 보호
    (2) 코드 변경 시 영향 범위 최소화
